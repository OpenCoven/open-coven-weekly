# Coven is not a harness.<br/>It's the *substrate*.

<div style="display:flex;flex-direction:column;gap:12px;max-width:760px;margin-top:1.5rem">
  <div class="card" style="display:flex;gap:12px;align-items:flex-start">

    <p><strong>Harnesses</strong> (Codex, Claude Code, and future adapters) bring their own credentials, models, and tool runtimes.</p>
  </div>
  <div class="card" style="display:flex;gap:12px;align-items:flex-start">
    <div style="width:6px;height:6px;border-radius:50%;background:#9A8ECD;flex-shrink:0;margin-top:8px"></div>
    <p><strong>Coven</strong> provides the shared layer: enforced project boundaries, supervised PTYs, append-only session records, and a stable local API.</p>
  </div>
  <div class="card" style="display:flex;gap:12px;align-items:flex-start">
    <div style="width:6px;height:6px;border-radius:50%;background:#9A8ECD;flex-shrink:0;margin-top:8px"></div>
    <p><strong>Any harness, one substrate.</strong> Consistent lifecycle, replay, and auditability regardless of which provider ran the work.</p>
  </div>

# The *OpenCoven* ecosystem.

<div style="display:grid;grid-template-columns:1fr 1fr;gap:16px;max-width:900px;margin-top:1.5rem">
  <div class="card">
    <div class="card-title">Coven Runtime</div>
    <p>Rust daemon. PTY lifecycle, session records, socket API, project boundary enforcement. The authority layer.</p>
  </div>
  <div class="card">
    <div class="card-title">CastCodes</div>
    <p>First-party cockpit. Sessions, panes, diffs, review, diagnostics. Cast Agent manages harness sessions inside it.</p>
  </div>
  <div class="card">
    <div class="card-title">OpenMeow</div>
    <p>macOS notch companion. Notch-as-inbox: toss work to Coven subagents without leaving your flow.</p>
  </div>
  <div class="card">
    <div class="card-title">OpenClaw Bridge</div>
    <p>Plugin that connects OpenClaw familiars (Cody, Kitty, Nova) into the Coven session substrate.</p>
