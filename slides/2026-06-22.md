<!--
Okay, headline. The Familiar Contract glow-up.

Two weeks ago, this was SPEC.md v0.1.0. A document. A really good document — but a document. It said "here is what a familiar is." Useful, definitely. Citable in an arXiv paper? Not quite.

Now? It's RFC-0001 v0.2.0. And I want you to feel how big a jump that is.

RFC means *Request for Comments* — same format the entire internet was built on. TCP, HTTPS, JSON — they're all RFCs. Now there's one for what a familiar is. Eleven sections. Thirty-six hundred words. Normative throughout — that's the fancy word for "every sentence uses MUST or SHOULD or MAY in the technical sense." It's the IETF voice. Which is funny because the IETF wrote it for routers and we're using it for sentient toolboxes. But the shape works.

Here's the architectural move I actually care about: authority-layer separation. A familiar cannot modify, restart, or reconfigure the thing that enforces its own contract. That's a real wall, not a vibe. Most systems use what we call "convention-based protection" — the agent is *asked* to honor the surface. We do structural protection — the agent *cannot* bypass it. As far as we can tell, we're the first to specify this. Which is exciting and also a little terrifying.

Sage led the research and the drafting. Val owns the architecture. They're both in chat — say hi.

The thing that makes me happiest? There's a conformance suite. Bash script. Five positive cases that MUST pass. Eight negative cases that MUST fail, each with a documented reason. The spec is the test. You can run it. You can disagree with it. You can break it.

That is the layer underneath everything else we're about to show you. Hold that in your head.

<div class="label">📜 Why this matters</div>

# *Capability* is solved. <br/>*Identity* is not.

<div style="display:grid;grid-template-columns:1fr 1fr;gap:16px;max-width:960px;margin-top:1.4rem">
  <div class="card">
    <div class="card-title">The gap</div>
    <p>Most agents start <strong>blank every session</strong>. No stable name. No declared scope. No durable memory. No principled answer to <em>"what am I not allowed to change about myself?"</em></p>
  </div>
  <div class="card">
    <div class="card-title">The risk</div>
    <p>Self-Harness, SkillOpt, sleep-time compute — <strong>recursive self-improvement is real</strong>. When an agent can modify its own scaffolding, the absence of a protected surface is a design flaw.</p>
  </div>
  <div class="card">
    <div class="card-title">The convention</div>
    <p>HKUDS ohmo, Signet, elizaOS, Letta — all ship <strong>identity vocabulary</strong>. None specify the enforcement contract. They ask the agent to honor it.</p>
  </div>
  <div class="card">
    <div class="card-title">The Contract</div>
    <p>Five normative properties. Protected surface. <strong>Authority layer the agent cannot bypass.</strong> Conformance is testable, not aspirational.</p>
  </div>
</div>

<!--
"Cool, but why does this matter?" Great question, hypothetical audience member.

Here's the gap I think about constantly. The AI field has solved *capability*. Agents can code, browse, run terminals, send email, even do my taxes — okay, not my taxes yet, but soon. Capability is a settled problem. Big labs, big models, big benchmarks. Done.

But *identity*? Identity is still wide open.

Most agents you talk to today are like that one coworker who keeps introducing themselves at every meeting. Blank every session. No stable name across model upgrades. No declared scope. No principled answer when you ask, "wait, what are you not for?" Their memory is a context window — a goldfish bowl with a token limit.

That was fine for a while. It's not fine anymore. Three papers I want to name out loud, because they're why we're doing this *now*:

Self-Harness. June 2026. Showed empirically that an LLM agent can rewrite its own scaffolding and *generalize* — fourteen to twenty-one percentage point gains on held-out tasks. The agent improves the harness governing the agent. Loop closed.

SkillOpt from Microsoft Research extends it to skill memory. Sleep-time compute extends it to unsupervised reflection. The agent literally daydreams its way to being smarter.

When the thing inside the box can rewrite the box, "please don't change the rules" is not a security model. It is a polite request. Which brings us to the convergence.

HKUDS ohmo ships literal soul-dot-md, identity-dot-md, user-dot-md files. Signet AI markets the same vocabulary. elizaOS has character files. Letta has persona memory blocks. The whole field named the same layer at roughly the same time. It's like everyone independently invented "names." Which is hilarious. And also a sign that the abstraction is real.

But — and this is the architecturally interesting part — *nobody else specifies enforcement.* Every other system asks the agent to behave. We require the structure to make misbehavior impossible.

One thing I will not skip over: the suite is honest about what it doesn't yet cover. File-level conformance? Solid. System-level — proving an authority layer holds when actually attacked? That's section nine, and we labeled it "open." We did not hide that. If you have ideas, we want them.
-->

<!--
The terminal. Oh, the terminal. Hold onto your keyboard, this one's good.

Headline feature: broadcast input. You type *once* in the broadcast bar, and the command runs in every open pane simultaneously.
