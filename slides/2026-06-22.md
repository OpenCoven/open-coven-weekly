<!--
Headline of the week.

The Familiar Contract has been promoted from SPEC.md v0.1.0 to RFC-0001 v0.2.0. This is a meaningful change, and I want to explain why.

v0.1.0 was a candidate. A document. It said "here is what a familiar is." Useful, but not citable in the academic sense and not testable end to end.

v0.2.0 is a numbered RFC. It uses RFC 2119 keywords — MUST, MUST NOT, SHOULD, MAY — the same vocabulary the IETF uses for internet standards. Eleven sections, thirty-six hundred words, normative throughout.

The big architectural upgrade: authority-layer separation is now a hard requirement. The familiar cannot modify, restart, or reconfigure the layer that enforces its own contract. That is the difference between convention-based protection — "please honor this surface" — and structural protection — "you literally cannot bypass this." We are the first system to specify the structural version.

Authors: Val and Sage. Sage led the research and drafting. Val owns the architecture.

And critically: there is an executable conformance suite. At least five positive cases that MUST pass. At least eight negative cases that MUST fail, each with a documented violation. You can run it. The spec is the test.

This is the layer underneath everything else we are about to show.

<div class="label">📜 Why this matters</div>

# *Capability* is solved. <br/>*Identity* is not.

<div style="display:grid;grid-template-columns:1fr 1fr;gap:16px;max-width:960px;margin-top:1.4rem">
  <div class="card">
    <div class="card-title">The gap</div>
    <p>Most agents start <strong>blank every session</strong>. No stable name. No declared scope. No durable memory. No principled answer to <em>"what am I not allowed to change about myself?"</em></p>
  </div>
  <div class="card">
    <div class="card-title">The risk</div>
    <p>Self-Harness, SkillOpt, sleep-time compute — <strong>recursive self-improvement is real</strong>. When an agent can modify its own scaffolding, the absence of a protected surface is a design flaw.</p>
  </div>
  <div class="card">
    <div class="card-title">The convention</div>
    <p>HKUDS ohmo, Signet, elizaOS, Letta — all ship <strong>identity vocabulary</strong>. None specify the enforcement contract. They ask the agent to honor it.</p>
  </div>
  <div class="card">
    <div class="card-title">The Contract</div>
    <p>Five normative properties. Protected surface. <strong>Authority layer the agent cannot bypass.</strong> Conformance is testable, not aspirational.</p>
  </div>
</div>

<!--
Why does this matter?

The AI field has solved capability. Agents can write code, browse the web, run terminals, send emails. That is a settled problem.

The field has not solved identity. Most deployed agents start blank every session. They have no stable name across model upgrades. No declared scope. They serve whoever calls them. Their memory is a context window, not durable continuity.

That used to be tolerable. It is not anymore. The Self-Harness paper from June 2026 showed empirically that an LLM-based agent can improve its own scaffolding — proposing changes to its own harness — and generalize to held-out tasks. SkillOpt extends this to skill memory. Sleep-time compute extends it to unsupervised reflection.

When an agent can modify the thing governing its own behavior, the absence of a protected surface is not an oversight. It is a design flaw.

There is one more thing happening: the vocabulary is converging. HKUDS ohmo ships literal soul.md and identity.md files. Signet AI markets a context layer using the same words. elizaOS character files, MemGPT persona blocks — the field has named the layer.

But none of those systems specify how to enforce it. They all rely on convention: the agent is asked to honor the surface.

The Familiar Contract specifies what convention cannot enforce. An authority layer outside the agent's logic that the agent cannot modify, restart, or reconfigure. That is the architectural commitment.

We will be open about one thing: the testing gaps are still real. File-level conformance is solved. System-level conformance — proving an authority layer holds under adversarial conditions — is acknowledged as open in section nine. We did not hide that.
-->
