<!--
PR #750: Daily Memory Context Injection.

Here's the problem this solves: every time Cave started a session, familiars were essentially waking up cold. No context about what happened yesterday, what was worked on, what decisions got made. Just: hello, I'm your familiar, what can I do for you?

Now, Cave injects the daily memory context on session startup. Familiars wake up knowing what happened. They're not starting from scratch every time.

This is one of the foundational promises of the familiar model — that your familiar persists, that it remembers. PR #750 is a big step toward actually delivering that promise.
-->

<!--
Okay — this is the late-breaking one. The architecture brainstorm: Plugins Out, Skills and Workflows and Roles In.

The conversation we kicked off this week: should we strip the plugin marketplace from coven-cave entirely?

The argument is this: a plugin marketplace implies that capability lives outside the familiar — you install a plugin to add a skill. But that's the wrong mental model. Familiars shouldn't need plugins. They should have trained, composable capability baked into their identity through Skills, Workflows, and Roles.

This is still in design. We're not announcing a decision tonight — we're opening the conversation. What do you think? If you have opinions on this — for or against — now is the time to share them.
-->
